Guía de Métodos para RestController

Integrantes: Javier Garcia, Armando Montiel, Leandro Rúa

"/api" (GET)

Descripción: Retorna un saludo simple "Hello, world!".

Keys: Ninguna

"/api/all/Users" (GET)

Descripción: Retorna una lista de todos los usuarios almacenados en la base de datos.

Keys: Ninguna

"/api/all/Notes" (GET)

Descripción: Retorna una lista de todas las notas almacenadas en la base de datos.

Keys: Ninguna

"/api/id/User" (GET)

Descripción: Retorna un usuario específico según el ID proporcionado.

Keys: id

"/api/id/Note" (GET)

Descripción: Retorna una nota específica según el ID proporcionado.

Keys: id

"/api/Update/User" (PUT)

Descripción: Actualiza la información de un usuario específico según el ID proporcionado.

Keys: username, email, password, id, token

"/api/Update/Note" (PUT)

Descripción: Actualiza la información de una nota específica según el ID proporcionado.

Keys: UserID, Title, Content, id, token

"/api/new/User" (POST)

Descripción: Inserta un nuevo usuario en la base de datos.

Keys: username, email, password

"/api/new/Note" (POST)

Descripción: Inserta una nueva nota en la base de datos.

Keys: UserID, Title, Content, token

"/api/login" (POST)

Descripción: Autentica a un usuario según el nombre de usuario y la contraseña proporcionados, y genera un token JWT para el usuario autenticado.

Keys: username, password

"/api/Delete/User" (DELETE)

Descripción: Elimina un usuario específico de la base de datos según el ID proporcionado.

Keys: id, token

"/api/Delete/Note" (DELETE)

Descripción: Elimina una nota específica de la base de datos según el ID proporcionado.

Keys: id, token
